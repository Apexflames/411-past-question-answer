b Systems Development Life Cycle (SDLC) 5 Marks

The Systems Development Life Cycle (SDLC) is a structured process used by organisations and analysts to plan, design, develop, test, and deliver information systems. It has 7 steps:

  1. Planning / Feasibility Study Define scope and objectives. Assess technical, financial, and operational feasibility. Key question: "Is this system worth building?"
  2. System Analysis Study the existing system in detail. Gather user requirements through interviews, questionnaires, and observation. Produce a requirements specification. Tools: DFDs, ERDs.
  3. System Design Design the new system — database structure, interfaces, and architecture. Covers Logical Design (what it should do) and Physical Design (how it will do it). Tools: Use Case and Class diagrams.
  4. Implementation / Coding Programmers write the actual code based on the design specification. This is where the system is physically built.
  5. Testing The system is tested against requirements — unit testing, integration testing, and user acceptance testing (UAT). Bugs are identified and corrected.
  6. Deployment / Installation The system goes live. Methods include direct changeover, parallel running, pilot conversion, or phased conversion.
  7. Maintenance The system is monitored after deployment. Bugs are fixed (corrective), improvements are added (perfective), and the system adapts to change (adaptive). This phase lasts the system's lifetime.

b System Analysis vs System Design — Definitions & Importance 6 Marks
📖 System Analysis
Definition: System Analysis is the process of studying and examining an existing system to understand how it works, identify its problems, and determine requirements for an improved or new system. It answers: "What should the system do?"

Key activities: Investigating through interviews, questionnaires, observation · Creating models (DFDs, ERDs) · Identifying problems and inefficiencies · Determining functional/non-functional requirements · Producing a requirements specification document
🛠️ System Design
Definition: System Design is the process of creating detailed technical specifications for the new system based on the requirements from analysis. It answers: "How will the system do it?" It has two phases: Logical Design (what the system should do) and Physical Design (how it will be built).

An Activity Diagram is a UML behavioral diagram that models the flow of activities (steps/actions) within a system. It shows the sequence of actions, decision points, parallel (concurrent) flows, and the beginning and end of a process.

🔧 How Activity Diagrams are Used in System Design
Document business processes BEFORE building the system so stakeholders can verify the logic · Model the flow of a specific use case step-by-step · Identify decision points (branches), parallel operations (fork/join), and exception paths · Help developers understand exactly what code logic is needed · Communicate system behaviour to non-technical stakeholders

b Five Classifications of Systems 7 Marks

A system is a set of interrelated components that work together to achieve a common objective. Systems can be classified in five ways:

1. Open vs. Closed System

Open: Interacts with its environment — receives inputs from and sends outputs to the outside world. Example: A university (students enter, graduates exit). Most IT systems are open.
Closed: Self-contained; no interaction with the environment. Example: A controlled laboratory experiment.

2. Natural vs. Artificial System

Natural: Exists without human intervention. Example: The human digestive system, the water cycle.
Artificial: Designed and built by humans. Example: A hospital management system, a banking system.

3. Deterministic vs. Probabilistic

Deterministic: Same inputs always produce the same outputs — fully predictable. Example: A calculator (5+3 = always 8).
Probabilistic: Outcomes subject to chance or probability. Example: A weather forecast, stock market analysis.

4. Static vs. Dynamic System

Static: Does not change over time; no memory of past states. Example: A printed map.
Dynamic: Changes over time and responds to its environment. Example: A traffic management system adjusting signals in real time.

5. Physical vs. Abstract System

Physical: Has tangible components you can see and touch. Example: A computer, a manufacturing plant, the human body.
Abstract: Made of ideas, concepts, or plans — no physical form. Example: A mathematical model, a software design plan, a business strategy.

Question Six
System Definition & Five Characteristics  ·  Library Management System Class Diagram

a Define a System & Explain Five Characteristics 7 Marks
📖 Definition of a System
A system is a set of interrelated and interdependent components (elements, subsystems, or parts) that work together in a coordinated manner to achieve a common goal or objective. Systems transform INPUTS into OUTPUTS within a defined BOUNDARY and operate within an ENVIRONMENT.

1. Components / Elements

Every system is made up of sub-systems or elements that each perform a specific function. Example: A hospital system has sub-systems — outpatient dept, pharmacy, laboratory, emergency unit, billing — each contributing to the whole.

2. Interrelationship / Interdependence

Components are connected and depend on each other. A change in one part affects other parts. Example: If a hospital's lab breaks down, diagnosis stalls, which delays treatment — the components are interdependent.

3. Boundary

Every system has a boundary separating it from its environment — defining what is INSIDE and what is OUTSIDE. Example: A library management system's boundary includes catalogue, members, and loans — but NOT the city council (outside).

4. Purpose / Objective

Every system exists to achieve a specific goal. All components work towards this common purpose. Example: A university attendance system's purpose is to track student attendance and flag students who may be barred from examinations.

5. Environment

Every system exists within an environment — external factors and entities that influence the system but are NOT part of it. Example: A banking system's environment includes customers, government regulators, and the economy — all influence the system from outside.
